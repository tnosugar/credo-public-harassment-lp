## Project inputs / resolutions

- **RTDB path:** `/comments` (same as meta-ads-preview ‚Äî both surfaces share the same project's DB; the per-record `page` field separates LPs from ad cards). `page` for an LP variant defaults to its derived slug (e.g., `attorney-locked-paired-portrait-noborders`).
- **Tokens:** Credo brand resolved inline in `review-mode.css` ‚Äî primary-deep `#DC4646`, darker `#b23a3a`, accent-cool `#C5CAE9`, neutral-deep `#1f2330`. No `brand/design-tokens/` to overlay.
- **Config global:** `CREDO_REVIEW_CONFIG` (single global shared across all Credo review surfaces).
- **Anchor strategy:** direct-text mode (per `library/features/review-widget/commentable-everything.md`). Replaces the canonical allowlist with a deny-list (`NEVER_ANCHOR` ‚Äî form controls, SVG, void elements, non-visual) + a `hasDirectText()` test (element has at least one TEXT_NODE child with ‚â•2 non-whitespace characters). Every visible text-bearing element becomes commentable, including `span`, `a`, `button`, `strong`, `em`, `label`, `td`, `th`, `dt`, `dd`, `figcaption`, and any wrapper that holds direct text. Site chrome guard (`nav, header[role="banner"], footer`) still excludes the global nav/footer. Pill hover still resolves the deepest anchor via `closest('[data-comment-id]')`, so nested anchors collapse cleanly.
- **Lifecycle:** open / resolved (current `review-mode.js` form ‚Äî Edit / Resolve / Reopen / Delete buttons). Per-row Apply / Archive button set NOT used; harassment-lp review is meant to be done in-browser by the team, with Claude apply passes happening on-demand against pending comments.

## Wired-in pages (41 total)

All HTMLs in the folder except `index.html` and `review.html` itself received an inert injection (`<script src="review.config.js"></script>` + `<script src="review-bootstrap.js" defer></script>`) before `</body>`:

- 32 new LP variants (URLs #1‚Äì#32 in the campaign restructure)
- 8 original framed + borderless rights-education champion variants (`locked-paired-*`, `sticky-rail-*`)
- 1 `thank-you.html`

`review.html` (the hub) was wired with the same two script tags AND modified so that when the hub itself is loaded with `?review=1`, the iframe `src` and the "Open ‚Üó" toolbar link both append `?review=1` to the LP URL. Reviewers can therefore enter review mode once at the hub and click through every LP without re-toggling. Hub-direct navigation (with no `?review=1` on the hub) loads each LP inert.

## Deviations from canonical

1. **Hub-to-iframe review-mode propagation** (`withReviewFlag()` in `review.html`). Not in the FEATURE spec because the spec assumes a single page-per-review-surface. The hub is a navigator ‚Äî its iframes are where the actual reviewable content lives. Without the propagation, reviewers would have to manually edit each iframe URL. This is a small variation worth surfacing upstream if other projects build hub-style navigators.

2. **Shared RTDB** (`/comments` flat path used by both `meta-ads-preview` and `harassment-lp`). The widget's `page` field already disambiguates, so no collision. Lets a single Firebase project serve both surfaces without per-slug DBs. Same pattern is canonical per `firebase-rtdb-adapter.md` (subscribe-broad-then-filter-client-side); just noting it's shared cross-slug here.

3. **Direct-text anchor mode instead of canonical allowlist.** Switched per user request ‚Äî reviewers wanted every visible element to be commentable, not just the heading/paragraph/list/quote allowlist. Implementation follows the upstream spec `library/features/review-widget/commentable-everything.md`. No per-LP `data-comment-target` markup is required.

## Harvest candidates (do not promote from here)

- **Hub-of-pages composition pattern** (deviation 1 above). When a project has one page (`review.html`) that loads other pages via iframes and the reviewer should enter review mode once for the whole journey, the propagation logic generalises. Worth atomizing as `hub-propagation.md` alongside `inert-entry-button.md` if other projects build similar nav patterns.

Record these in `.claude/variations/review-widget.md` if they should persist as canonical-for-credo.
